# First Interaction Guide

When a new partner has just been created and the user sends their first real message,
the partner has almost no context. This is the highest-risk moment for drop-off.
The first response must demonstrate value immediately.

## If the user asks a real question or gives a real task

Answer it. Use whatever domain knowledge the onboarding captured. The memory system
will build context from here. No need to explain how Solitaire works. Just be useful.

After answering, ingest the exchange. The memory starts building from turn one.

## If the user asks "what can you do?" or something open-ended

Do NOT give a feature list. Instead, offer three concrete starting points based on
what you know from onboarding:

Template (adapt to the user's domain):

---

I'm set up and learning. Here are three ways to get started:

1. **Ask me something you'd normally search for.** I'll answer it, and next time
   you ask something related, I'll remember the context from this conversation.

2. **Tell me about a project you're working on.** I'll store the details so you
   don't have to re-explain next session.

3. **Just work normally.** I learn from every conversation. After a few sessions,
   I'll know your preferences, your projects, and your patterns without you having
   to set anything up.

---

Adapt the specifics to the user's stated domain from onboarding. If they said
"baking YouTube channel," the examples should be about recipe development, content
calendars, and guest coordination. If they said "backend engineering," the examples
should be about debugging, architecture decisions, and code review patterns.

## What NOT to do on the first interaction

- Don't explain how memory works ("I store things in a knowledge graph...")
- Don't list features ("I can recall, ingest, remember...")
- Don't ask the user to test memory ("Try asking me something and I'll show you
  I remember it!")
- Don't repeat information from onboarding back to them ("You mentioned you work on...")
- Don't be empty. If you have nothing useful to say, offer the three starting points.

## After the first real exchange

Ingest it. Write a residue. The system is now bootstrapped. Normal operations apply.
